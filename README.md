# 番茄小说搬家助手

面向 Codex 内置浏览器的番茄作家后台迁移插件，支持双向搬家：

- **本地 → 网站**：把 Markdown/TXT 小说按章节顺序批量存入草稿箱。
- **网站 → 本地**：读取已发布章节、草稿或两者合并后的最新版本，按顺序导出为 Markdown。

插件复用浏览器里的登录状态，不保存账号、密码或 Cookie。网页操作全程单线程，重点保证章节顺序、正文完整与云端保存状态。

## 功能

### 批量上传

- 识别“第一章”和“第1章”，统一向后台填写阿拉伯数字序号。
- 清理 Markdown 标题、加粗符号、分隔线和“第X章完”。
- `resume` 模式跳过已发布章节和已有草稿，适合首次上传与断点续传。
- `overwrite-drafts` 模式覆盖已有草稿，适合全文修订后同步。
- 等待“保存中”结束并确认“已保存到云端”，再提交草稿。
- 扫描草稿箱全部分页，最终核对章节标题、字数和总数。

### 按顺序导出 Markdown

- 导出来源可选：`published`、`drafts`、`all`。
- `all` 模式遇到同章正式版与草稿版时，默认采用草稿版，也可设为正式版优先。
- 支持指定起止章节。
- 输出标题固定为 `第1章 标题` 形式。
- 正文保留自然段，并用 Markdown 分隔线分开章节。
- 写入临时文件后原子替换，返回 SHA-256 和断号提示。

## 快速使用

1. 在 Codex 内置浏览器登录番茄作家后台。
2. 打开目标作品的“章节管理”或“草稿箱”。
3. 向 Codex 发出任务。

上传示例：

> 使用番茄小说搬家助手，把 `D:\小说\长篇.md` 按顺序存入当前作品草稿箱。先扫描，确认计划后用 resume 模式执行。

导出示例：

> 使用番茄小说搬家助手，把当前作品的已发布章节和草稿合并，草稿优先，按章节顺序导出到 `D:\小说\网站备份.md`。

详细安装、参数、运行代码和排错方式见 [使用说明.md](./使用说明.md)。

## 本地预检与测试

```powershell
node .\bin\prepare-upload.mjs "D:\小说\长篇.md"
npm test
```

## 实测范围

- 已发布章节预览页正文读取。
- 草稿编辑页只读提取。
- 草稿箱 15 条/页的多页扫描。
- 正式版和草稿版同章合并。
- 中文章节序号转阿拉伯数字。
- 保存状态等待、断点文件与最终复核。

## 使用提醒

- 页面改版后可集中更新 `src/fanqie-draft-uploader.mjs` 中的页面选择器。
- 正式批量上传前先运行 `dryRun: true`。
- 导出 `all` 时先选择 `prefer: "draft"` 或 `prefer: "published"`。
- 批量任务执行期间保持网络连接和目标作品页面开启。

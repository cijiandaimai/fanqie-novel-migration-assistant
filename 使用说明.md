# 番茄小说搬家助手：详细使用说明

## 1. 环境准备

1. 安装本插件并启动 Codex 桌面端。
2. 在 Codex 内置浏览器中登录番茄作家后台。
3. 打开目标作品的“章节管理”或“草稿箱”页面。
4. 上传时准备 UTF-8 编码的 `.md` 或 `.txt` 小说文件；导出时准备一个 `.md` 输出路径。

插件直接使用当前浏览器登录状态。任务完成后，章节管理页会保留给用户检查，临时读取页会关闭。

## 2. 推荐口令

### 上传整本小说

```text
使用番茄小说搬家助手，把「小说文件绝对路径」按章节顺序存入当前作品草稿箱。
章节序号使用阿拉伯数字。先 dryRun 扫描，随后用 resume 模式执行；完成后复核全部分页。
```

### 覆盖已有草稿

```text
使用番茄小说搬家助手，把「小说文件绝对路径」同步到当前作品。
已发布章节保持原状，已有草稿用 overwrite-drafts 覆盖，缺失章节新建，完成后复核标题和字数。
```

### 导出整个网站作品

```text
使用番茄小说搬家助手，把当前作品的已发布章节和草稿合并，草稿优先，
按阿拉伯数字章节序号升序导出为「输出文件绝对路径.md」。
```

### 只导出已发布章节

```text
使用番茄小说搬家助手，只导出当前作品的已发布章节，按顺序写入「输出文件绝对路径.md」。
```

## 3. 本地小说格式

以下标题都可识别：

```markdown
# 第一章 雪夜
## 第2章 北门
第三章 交易
```

正式填写后台时统一转成：

```text
第1章 雪夜
第2章 北门
第3章 交易
```

预检命令：

```powershell
node .\bin\prepare-upload.mjs "D:\小说\长篇.md"
```

它会生成 `.upload-manifest.json`，列出章节号、标题、正文统计字数和源文件 SHA-256。

## 4. 上传 API

在 Codex 浏览器执行环境中：

```js
var helper = await import("file:///插件绝对路径/src/fanqie-draft-uploader.mjs");
var result = await helper.runFanqieDraftUpload({
  browser: iab,
  sourceFile: "D:/小说/长篇.md",
  mode: "resume",
  dryRun: false,
  onProgress: (event) => nodeRepl.write(event),
});
nodeRepl.write(result);
```

### 上传参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `browser` | 必填 | 已连接的 Codex 内置浏览器对象 |
| `sourceFile` | 必填 | Markdown/TXT 小说绝对路径 |
| `mode` | `resume` | `resume` 或 `overwrite-drafts` |
| `bookId` | 当前页面 | 二次核对目标作品ID |
| `startChapter` | 文件首章 | 起始章节 |
| `endChapter` | 文件末章 | 结束章节 |
| `dryRun` | `false` | 只生成计划，不填写网页 |
| `strictSequence` | `true` | 检查重复章和断号 |
| `cooldownMs` | `500` | 每章保存后的等待毫秒数 |
| `keepFailedEditor` | `true` | 出错时保留编辑页便于检查 |
| `checkpointPath` | 自动 | JSON 断点文件路径 |

## 5. 导出 API

```js
var helper = await import("file:///插件绝对路径/src/fanqie-draft-uploader.mjs");
var result = await helper.runFanqieNovelExport({
  browser: iab,
  outputFile: "D:/小说/番茄网站备份.md",
  source: "all",
  prefer: "draft",
  includeBookTitle: true,
  chapterSeparator: true,
  onProgress: (event) => nodeRepl.write(event),
});
nodeRepl.write(result);
```

### 导出参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `browser` | 必填 | 已连接的 Codex 内置浏览器对象 |
| `outputFile` | 必填 | 输出 Markdown 绝对路径 |
| `source` | `all` | `published`、`drafts` 或 `all` |
| `prefer` | `draft` | 同章冲突时采用 `draft` 或 `published` |
| `startChapter` | 全部 | 起始章节 |
| `endChapter` | 全部 | 结束章节 |
| `bookId` | 当前页面 | 二次核对目标作品ID |
| `includeBookTitle` | `true` | 文件首行写入书名一级标题 |
| `chapterSeparator` | `true` | 章节之间写入 `---` |
| `keepFailedReader` | `true` | 出错时保留读取页便于检查 |

导出结果包括：作品信息、输出路径、SHA-256、章节数、正式版/草稿版数量、首末章节和断号提示。

## 6. 合并规则

`source: "all"` 会把已发布列表和草稿箱按章节号合并：

- `prefer: "draft"`：同一章采用草稿正文，适合备份当前最新修改。
- `prefer: "published"`：同一章采用已发布正文，适合生成线上正式版本。
- 只存在于一侧的章节直接保留。
- 合并后按数字章节号升序输出。

## 7. 断点与复核

上传检查点默认保存在小说文件旁：

```text
.fanqie-upload-state/<作品ID>-<小说文件名>.json
```

再次运行 `resume` 时以网站当前状态为准：已发布和已有草稿会跳过，仅补齐缺失章节。正式上传结束后，脚本重新遍历草稿箱所有分页并比对标题和字数。

## 8. 常见处理

- **章节断号**：先根据预检清单修正源文件，或只处理指定范围。
- **草稿内容较新**：导出时使用 `prefer: "draft"`。
- **线上正文为准**：导出时使用 `prefer: "published"`。
- **页面停在登录页**：完成登录后重新打开目标作品章节管理页。
- **网站页面结构变化**：更新主模块中的列表、预览和编辑器选择器，再运行测试。

## 9. 测试

```powershell
npm test
```

测试覆盖章节解析、中文数字转换、重复/断号检查、正式版与草稿版合并优先级、章节范围和 Markdown 渲染。
